Overview

The Proof‑of‑Contribution Protocol provides a unified, verifiable way to describe contributions across any project or ecosystem. It defines a canonical schema (pow.yaml), a deterministic validation pipeline, and a set of guarantees that ensure contributions are structured, portable, and machine‑verifiable.

Protocol Specification

The protocol defines:

  • Contribution data model
  • Contributor identity structure
  • Evidence types and formats
  • Validation rules and invariants
  • Versioning and forward‑compatibility guarantees

Full specification is available in the repository under /docs/spec.md.

pow.yaml Specification

pow.yaml is the canonical representation of contributions. It includes:

  • Project metadata
  • Contributor identities
  • Contribution entries
  • Evidence records
  • Document metadata

See /docs/pow-yaml-spec.md for the full field‑by‑field breakdown.

Validation Pipeline

The validation engine performs:

  1. Schema Validation — structure & types
  2. Semantic Validation — logical consistency
  3. Reference Validation — evidence URIs
  4. Integrity Validation — hashes & artifacts

See /docs/validation-pipeline.md for details.
